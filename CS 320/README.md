## Reflection

### How can I ensure that my code, program, or software is functional and secure?
To ensure that my code is functional, I rely on unit testing and validation. By writing JUnit tests, I can verify that each method behaves as expected and handles both normal and edge cases correctly. This helps catch errors early before they become bigger issues.

For security, I focus on validating inputs and enforcing constraints within my classes. For example, making sure fields are not null, limiting string lengths, and preventing invalid data from being stored. Writing tests for these conditions also helps confirm that the program behaves safely under different scenarios.

### How do I interpret user needs and incorporate them into a program?
I start by carefully reviewing the requirements and breaking them down into clear rules that the program must follow. For example, if a requirement states that a contact ID must be unique and within a certain length, I translate that into logic within the class and service methods.

I also consider how the user will interact with the system and what could go wrong. This helps me design test cases that reflect real-world usage. By aligning my code and tests with the requirements, I can ensure the program meets the user’s needs.

### How do I approach designing software?
When designing software, I begin by planning out the structure of the program, including the classes and their responsibilities. I try to keep each class focused on a single purpose, which makes the code easier to understand and maintain.

I also design with testing in mind. This means writing code in a way that can be easily tested, such as separating logic into methods and avoiding unnecessary complexity. Once the structure is in place, I implement the code step by step and continuously test it to make sure everything works correctly.

Overall, I focus on keeping the design simple, following the requirements closely, and using testing as a tool to guide development and ensure quality.
