## Reflection on Projects

### 1. What was the problem you were solving in the projects for this course?
In Project One, the task was to analyze the run-time and memory usage of different data structures used to store course information. This helped me understand how data structures impact program performance.  

In Project Two, the goal was to write a program that sorts and prints all courses in the Computer Science program in alphanumeric order. This required organizing and displaying data efficiently and correctly.

### 2. How did you approach the problem? Consider why data structures are important to understand.
I approached the projects by first analyzing the type of data I needed to store and how it would be used. For Project One, I tested arrays, vectors, and linked lists to compare their run-time performance and memory usage, which helped me see the practical impact of choosing the right structure. For Project Two, I used a vector to store course data and implemented sorting logic to print the courses in alphanumeric order.  

Understanding data structures was important because the efficiency of my program depended on selecting the right structure for the task. Knowing the trade-offs allowed me to design solutions that were not only correct but also efficient and scalable.

### 3. How did you overcome any roadblocks you encountered while going through the activities or project?
I initially struggled with understanding trade-offs between memory usage and speed among different data structures. I overcame this by testing each structure with sample data, recording the results, and debugging the sorting algorithm to ensure correct output for Project Two.

### 4. How has your work on this project expanded your approach to designing software and developing programs?
These projects reinforced the importance of choosing the right data structures and planning the algorithm carefully. I learned to approach software design more methodically, breaking problems into smaller, manageable parts while considering efficiency and readability.

### 5. How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
I have improved in writing modular and well-commented code, making it easier to read, debug, and adapt. Reflecting on my decisions also helps me document my reasoning, which will be useful for future coursework or professional projects.
